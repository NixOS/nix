# Name

`nix-store --register-validity` - register store paths as valid

## Synopsis

`nix-store` `--register-validity` [`--reregister`] [`--hash-given`]

## Description

The operation `--register-validity` reads serialised validity
information for store paths from standard input and registers those
paths as valid in the local Nix store.

The input consists of a sequence of records, one per store path. Each
record has the following format:

- path
- nar-hash
- nar-size
- deriver
- number-of-references
- reference-1
- ...
- reference-n

Here, *path* is the store path to register. *nar-hash* is the SHA-256
hash of the path’s NAR serialisation, and *nar-size* is the size of the
NAR in bytes. *deriver* is the store path of the derivation that
produced this path, or an empty line if unknown. *number-of-references*
specifies how many reference paths follow, which are listed on the
subsequent lines.

The input is typically generated by other Nix operations such as
`exportReferencesGraph`. It is the responsibility of the caller to
provide a complete closure.

If `--reregister` is specified, paths are registered even if they are
already marked as valid in the store.

If `--hash-given` is not specified, Nix recomputes the NAR hash and size
before registering the paths.

This is a low-level operation intended for internal use.
